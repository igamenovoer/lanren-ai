# Custom Development Tools

## Purpose

This directory houses custom scripts, utilities, and development aids specific to the Lanren AI project workflow.

## Contents

- Build automation scripts
- Testing utilities
- Code generation tools
- Data processing scripts
- Development environment helpers
- CI/CD utilities
- Custom linters or validators

## Organization

- Group related tools in subdirectories
- Include usage documentation with each tool
- Provide examples where applicable
- Document dependencies and requirements

## Best Practices

- Keep tools focused on single responsibilities
- Include help/usage information
- Test tools before committing
- Document any configuration needed
- Version tools appropriately

## Documentation

Each tool should have:
- Purpose and use case
- Installation/setup instructions
- Usage examples
- Required dependencies
- Configuration options
- Troubleshooting notes

## Document Format

Every documentation file should start with a HEADER section containing:
- **Purpose**: What this document is for
- **Status**: Current state (active/completed/deprecated/failed)
- **Date**: When created or last updated
- **Dependencies**: What this relates to or requires
- **Target**: Intended audience (AI assistants, developers, etc.)
