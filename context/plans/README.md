# Implementation Plans

## Purpose

This directory documents implementation strategies, feature roadmaps, and multi-step development plans. Break down complex features into manageable tasks.

## Contents

- Feature implementation roadmaps
- Migration strategies
- Refactoring plans
- Architecture evolution plans
- Version release plans
- Integration strategies

## Naming Convention

Use feature or system names:
- `api-redesign-roadmap.md`
- `database-migration-strategy.md`
- `v2-implementation-plan.md`

## Best Practices

- Break down complex work into phases
- Identify dependencies between tasks
- Include success criteria
- Document alternative approaches considered
- Update as plans evolve

## Document Format

Every document should start with a HEADER section containing:
- **Purpose**: What this document is for
- **Status**: Current state (active/completed/deprecated/failed)
- **Date**: When created or last updated
- **Dependencies**: What this relates to or requires
- **Target**: Intended audience (AI assistants, developers, etc.)
