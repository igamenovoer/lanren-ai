# Knowledge Summaries

## Purpose

This directory maintains analysis documents, project knowledge summaries, and consolidated findings from research or implementation work.

## Contents

- Library and tool comparison analyses
- Architecture decision rationales
- Implementation lessons learned
- Research findings
- Technology evaluations
- Performance analysis results
- Code review summaries

## Naming Convention

Use descriptive analysis topics:
- `library-comparison-analysis.md`
- `architecture-decision-rationale.md`
- `implementation-lessons-learned.md`

## Best Practices

- Keep summaries current and accurate
- Include supporting data and evidence
- Reference source materials
- Update when new information emerges
- Remove outdated summaries or mark as deprecated

## Document Format

Every document should start with a HEADER section containing:
- **Purpose**: What this document is for
- **Status**: Current state (active/completed/deprecated/failed)
- **Date**: When created or last updated
- **Dependencies**: What this relates to or requires
- **Target**: Intended audience (AI assistants, developers, etc.)
